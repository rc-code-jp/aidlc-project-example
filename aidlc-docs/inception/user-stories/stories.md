# User Stories

## Story Organization
- **Approach**: User Journey-Based
- **Primary Persona**: 一般利用者

## US-01 気持ちよく画像を選択できる
**As a** 一般利用者  
**I want** 分かりやすいアップロード導線から画像を選び、送信前に内容を確認したい  
**So that** 間違いのない画像を安心して判定に出せる

### Acceptance Criteria
- 画面に視認性の高いアップロード導線が表示されている
- 利用者は 1 枚の画像ファイルをファイル選択またはドラッグアンドドロップで選べる
- 画像選択後、送信前にプレビューが表示される
- 判定開始操作は選択済み画像に対して分かりやすく提供される
- 不正な入力や画像未選択時は判定処理を開始しない

### INVEST Notes
- **Independent**: 判定前のアップロード体験に限定
- **Valuable**: 利用者が安心して最初の操作を進められる
- **Testable**: 選択方法、プレビュー表示、送信可能状態を確認可能

## US-02 判定中の進行を迷わず理解できる
**As a** 一般利用者  
**I want** 判定中であることを視覚的に把握したい  
**So that** 処理が止まっているのか進んでいるのか迷わず待てる

### Acceptance Criteria
- 画像送信後、画面に判定中であることが視覚的に分かる表示が出る
- 判定中は現在進行中であることがテキスト以外の視覚要素でも認識できる
- 判定完了後は判定中表示が自然に結果表示へ切り替わる

### INVEST Notes
- **Independent**: 判定中の状態表現に集中
- **Valuable**: 待機中の不安を減らせる
- **Testable**: API リクエスト中の視覚状態変化を検証可能

## US-03 判定結果をひと目で理解できる
**As a** 一般利用者  
**I want** 犬らしさの結果を視覚的に強調された形で見たい  
**So that** 数値と判定内容をすぐに理解できる

### Acceptance Criteria
- 判定成功時に 0 から 100 のパーセントが視覚的に強調表示される
- 判定成功時に `犬`、`犬っぽい`、`犬ではない` のいずれかが見やすく表示される
- 結果表示は単なるテキスト列ではなく、情報の優先度が伝わるレイアウトになっている
- 犬が写っていると Gemini が判断した場合は 100% が表示される
- 犬っぽい対象の場合は 100% 未満の値が表示される
- 別画像をアップロードした場合は前回結果を新しい結果で更新する
- 判定理由は主結果を邪魔せず補足情報として読める

### INVEST Notes
- **Independent**: 成功結果の理解しやすさに限定
- **Valuable**: 本機能の中心価値をより伝わりやすくする
- **Testable**: API 応答に応じた強調表示と情報配置を確認可能

## US-04 失敗してもすぐ立て直せる
**As a** 一般利用者  
**I want** 判定に失敗したときも状況を理解して再試行したい  
**So that** 一時的な失敗でも操作をやり直して利用を継続できる

### Acceptance Criteria
- 外部 API 呼び出し失敗時に認識しやすいエラーメッセージが表示される
- エラー表示中でも利用者は選択済み画像や次の操作を把握できる
- 利用者は再度画像を選択または再送してやり直せる
- エラー表示中でも API キーや内部情報は画面に露出しない

### INVEST Notes
- **Independent**: 失敗時の復帰体験に限定
- **Valuable**: 失敗しても使う気持ちを維持できる
- **Testable**: エラー時の視認性と再試行導線を確認可能
