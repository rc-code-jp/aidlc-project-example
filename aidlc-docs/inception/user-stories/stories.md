# User Stories

## Story Organization
- **Approach**: User Journey-Based
- **Primary Persona**: 一般利用者

## US-01 画像を選択して判定を開始する
**As a** 一般利用者  
**I want** 画面から 1 枚の画像を選択して判定を開始したい  
**So that** 犬が写っているかをすぐ確認できる

### Acceptance Criteria
- 画面に画像アップロード操作が表示されている
- 利用者は 1 枚の画像ファイルを選択できる
- 画像選択後、判定開始操作または自動送信で判定フローへ進める
- 不正な入力や画像未選択時は判定処理を開始しない

### INVEST Notes
- **Independent**: 画像入力開始の責務に限定
- **Valuable**: 利用者が最初の操作を実行できる
- **Testable**: 画像選択と送信開始の可否を確認可能

## US-02 判定中の状態を把握する
**As a** 一般利用者  
**I want** 判定中であることを画面上で確認したい  
**So that** 処理が進んでいることを理解して待てる

### Acceptance Criteria
- 画像送信後、画面に判定中であることが表示される
- 判定中は利用者が現在処理待ちであると認識できる
- 判定完了後は判定中表示が結果表示へ切り替わる

### INVEST Notes
- **Independent**: ローディング状態の表現に集中
- **Valuable**: 利用者の不安を減らせる
- **Testable**: API リクエスト中の UI 状態を検証可能

## US-03 犬らしさの結果を理解する
**As a** 一般利用者  
**I want** 犬らしさのパーセントとラベルを見たい  
**So that** 画像に犬が写っている可能性を判断できる

### Acceptance Criteria
- 判定成功時に 0 から 100 のパーセントが表示される
- 判定成功時に `犬`、`犬っぽい`、`犬ではない` のいずれかが表示される
- 犬が写っていると Gemini が判断した場合は 100% が表示される
- 犬っぽい対象の場合は 100% 未満の値が表示される
- 別画像をアップロードした場合は前回結果を新しい結果で更新する

### INVEST Notes
- **Independent**: 成功結果表示の責務に限定
- **Valuable**: 本機能の中心価値を提供する
- **Testable**: API 応答に応じた表示結果を確認可能

## US-04 エラー時に再試行できる
**As a** 一般利用者  
**I want** 判定に失敗したとき理由が分かり再試行したい  
**So that** 一時的な失敗でも利用を継続できる

### Acceptance Criteria
- 外部 API 呼び出し失敗時にエラーメッセージが表示される
- 利用者は再度画像を選択または再送してやり直せる
- エラー表示中でも API キーや内部情報は画面に露出しない

### INVEST Notes
- **Independent**: 失敗系シナリオに限定
- **Valuable**: 失敗時も利用を継続できる
- **Testable**: API エラー時の表示と再試行動作を確認可能

## Persona Mapping
- 一般利用者: US-01, US-02, US-03, US-04
