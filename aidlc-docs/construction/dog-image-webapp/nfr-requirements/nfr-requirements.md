# NFR Requirements - dog-image-webapp

## Summary
- 初期版はローカル実行前提の Web アプリとして提供する
- 応答時間は Gemini API 呼び出しを含めて数秒かかることを許容する
- 外部 API 失敗時は利用者へエラーメッセージを表示し、再試行可能にする
- 基本的なアクセシビリティ対応を初期版から含める

## Performance
- 厳密な秒数 SLA は設けない
- 利用者が待機状態を理解できるよう、判定中表示を必須とする
- 同時多数ユーザー向け最適化は初期版では対象外とする

## Security
- Gemini API キーはサーバー側環境変数で管理する
- ブラウザへ API キーや内部エラー詳細を露出しない
- 画像は永続保存せず、リクエスト処理の範囲内だけで扱う

## Reliability
- Gemini API の失敗時は利用者へ失敗を通知する
- 利用者が同一画面から再試行できること
- 初期版では自動リトライや外部監視基盤は必須としない

## Maintainability
- Next.js 内でも UI、API Route、Gemini 連携ロジック、設定読込を分離する
- プロンプトやレスポンス正規化ロジックは専用モジュールへ集約する
- 主要処理はテストしやすい純粋関数または小さなサービス単位に分ける

## Usability and Accessibility
- 画像アップロード操作はラベル付きで提供する
- 判定状態、成功、失敗を画面上で判別できること
- キーボード操作と基本的なスクリーンリーダー配慮を行う
- 色だけに依存せず、テキストでも状態と結果を伝える
